¿Qué es un recurso frontend y cuál es su función en el desarrollo web?

Un recurso frontend se refiere a la parte de una aplicación web o sitio web con la que el usuario interactúa
directamente en su navegador. Esto incluye todo lo que el usuario ve, escucha y
con lo que interactúa en la página web, como textos, imágenes, videos y formularios. La función del desarrollo
frontend es convertir los datos y la lógica de la aplicación en una interfaz gráfica, utilizando para ello HTML,
CSS, y JavaScript, de manera que los usuarios puedan interactuar eficientemente con la aplicación o sitio web.

¿Cuáles son las principales tecnologías o herramientas utilizadas en el desarrollo frontend y por qué son importantes?

Las principales tecnologías utilizadas en el desarrollo frontend son:
HTML: Para estructurar el contenido de la web.
CSS: Para estilizar y presentar el contenido, incluyendo el diseño, colores y fuentes.
JavaScript: Para añadir interactividad y dinamismo a la página web.

Describe el concepto de responsividad en el diseño web y menciona al menos dos técnicas para lograrlo.

El diseño responsivo es una práctica de diseño web que busca asegurar que una página
web se vea bien y funcione correctamente en una variedad de dispositivos y tamaños de pantalla.
Dos técnicas para lograr un diseño responsivo son:

Media Queries en CSS: Permiten aplicar estilos diferentes dependiendo del tamaño de la pantalla o características del dispositivo.

Diseño Flexible: Utiliza unidades de medida relativas (como porcentajes) para los elementos
de la página, permitiendo que se adapten a diferentes tamaños de pantalla.

¿Cuál es la diferencia entre HTML, CSS y JavaScript en el contexto del desarrollo frontend? Explique brevemente el papel de cada uno de estos lenguajes.

HTML: Es el esqueleto de la página web, utilizado para estructurar el contenido.

CSS: Define el estilo de la página, incluyendo colores, fuentes y layout.

JavaScript: Añade funcionalidad e interactividad, permitiendo modificar dinámicamente el contenido y el estilo.

Explora el concepto de frameworks frontend y proporciona ejemplos de al menos dos de ellos. ¿Por qué los desarrolladores utilizan frameworks en lugar de escribir todo el código desde cero?

Los frameworks frontend son conjuntos de herramientas y bibliotecas que proporcionan una estructura básica
para el desarrollo de aplicaciones web, facilitando tareas comunes y mejorando la productividad. Ejemplos incluyen:

React: Permite construir interfaces de usuario de manera eficiente.

Angular: Ofrece una solución completa para el desarrollo de aplicaciones web SPA.

Los desarrolladores utilizan frameworks porque ofrecen componentes reutilizables, convenciones y mejores prácticas integradas,
lo que reduce el tiempo de desarrollo y mejora la calidad y mantenibilidad del código.

En el contexto del desarrollo frontend, ¿cuál es la diferencia entre un CMS (Sistema de Gestión de Contenidos) y un framework? Proporciona ejemplos de ambos.

CMS (Sistema de Gestión de Contenidos): Es una plataforma que permite a los usuarios crear, gestionar y modificar contenido
en un sitio web sin la necesidad de tener conocimientos especializados en desarrollo web. Ejemplos: WordPress, Drupal.

Framework: Es una colección de herramientas y bibliotecas que proporciona una estructura para desarrollar aplicaciones web de
forma más eficiente, exigiendo a los desarrolladores escribir código para implementar funcionalidades específicas. Ejemplos: React, Angular.

La principal diferencia radica en su propósito y en cómo se utilizan para construir y administrar sitios web y aplicaciones web.
